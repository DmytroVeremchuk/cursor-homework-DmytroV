Список основних команд

  • git config --global user.name blabla
  • git config --global user.email blabal@bla

    • Перше, що вам слід зробити після встановлення Git — вказати ваше ім'я та адресу електронної пошти. Це важливо, тому що кожен коміт у Git містить цю інформацію, і вона включена в коміти, що передаються вами, і не може бути далі змінено. Знову ж таки, якщо вказано опцію --global, то ці налаштування достатньо зробити лише один раз, оскільки в цьому випадку Git використовуватиме ці дані для всього, що ви робите в цій системі.

  • git init - "Init" - це скорочення від "initialize". Це означає, що коли Ви пишете "git init", Git "включається" або "запускається" для цього репозиторію.
  • git status - Команда git status відображає стан робочого каталогу та розділу проіндексованих файлів. З її допомогою можна перевірити індексацію змін та побачити файли, які не відстежуються Git
  • git add - Це перша команда в ланцюжку операцій, що наказує Git «зберегти» знімок поточного стану проекту історії коммітів. Коли git add використовується як окрема команда, вона переносить очікуючі зміни з робочого каталогу розділ проіндексованих файлів
  • git commit - Це операція, яка бере всі підготовлені зміни та відправляє їх у репозиторій як єдине ціле
  • git log - За умовчанням (без аргументів) git log перераховує комміти, зроблені в репозиторії у зворотному напрямку хронологічному порядку – останні коміти знаходяться вгорі
  • git checkout - Команда git checkout дозволяє переміщатися між гілками, створеними командою git branch. При перемиканні гілки відбувається оновлення файлів у робочому каталозі відповідно до версії, що зберігається в цій гілці, а Git починає записувати нові комміти в цій гілці
  • git branch - Це команда для управління гілками у репозиторії Git. Гілка в Git'і - це просто "ковзкий" покажчик на один із коммітів. Коли ви створюєте нові комміти, вказівник гілки автоматично зсувається вперед, знову створеному комітету
  • git push - Команда git push використовується для вивантаження вмісту локального репозиторію у віддалений репозиторій. Вона дозволяє передати коміти з локального репозиторію у віддалений
  • git pull - Команда git pull використовується для вилучення та завантаження вмісту з віддаленого репозиторію та негайного оновлення локального репозиторію цим вмістом. Злиття видалених вищих змін у локальний репозиторій - це звичайне завдання робочого процесу, що виникає при спільній роботі на основі системи Git
  • git merge - Злиття використовується в Git, щоб зібрати воєдино розгалужену історію. Команда git merge виконує злиття окремих напрямів розробки, створених за допомогою команди git branch в єдину гілку
  • git clone - Це утиліта командного рядка Git для вибору існуючого репозиторію та створення його клону, тобто копії

    • Advanced

  • git revert - Це операція для безпечного скасування змін, дія якої спрямована у майбутнє. Для відкату змін команда не видаляє з історії комміти чи батьківські елементи, а створює новий коміт зі скасуванням потрібних дій
  • git reset - Це команда, яка використовується для скасування локальних змін у репозиторії Git
  • git rebase - Ще один спосіб перенести зміни з однієї гілки до іншої. Rebase стискає всі зміни в один патч. Потім він інтегрує патч у цільову гілку
  • git cherry-pick - Команда git cherry-pick бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту у поточній гілці. Ця можливість корисна в ситуації, коли потрібно забрати парочку коммітів з інший гілки, а чи не зливати гілку цілком із усіма внесеними у ній змінами.
  • git stash - Команда git stash дозволяє на якийсь час «здати в архів» (або відкласти) зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше
